## Search: Search for relevant information + best practices

Best practices 2025-2026: when migrating off a vendored/forked SDK, switch package.json to '@modelcontextprotocol/sdk': '^x.y.z', npm install, test subpath imports (@modelcontextprotocol/sdk/server/mcp.js) resolve, verify lockfile/SBOM accuracy. For preserving a small local patch: patch-package (npm i -D patch-package; edit node_modules; npx patch-package <pkg>; commit patches/<pkg>.patch; postinstall: patch-package). CRITICAL CAVEAT for PUBLISHED/distributed packages (dokoro IS published as dokoro with bin entries): patch-package only patches the MAINTAINER's local node_modules, NOT consumers who install the package — the SDK arrives transitively unpatched for end users. So the requestId-optional patch would NOT reach real users via patch-package. Recommended: remove postinstall for distributed CLIs (CI breakage risk); prefer dropping the patch or handling in OWN userland code. patch-package last updated 2023; pnpm/yarn/bun have native patch support. moduleResolution node16/nodenext + ESM subpath exports generally fine, watch for resolution quirks.
